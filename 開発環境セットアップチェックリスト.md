# 開発環境セットアップチェックリスト

## 前提条件の確認

### 必須ソフトウェア

- [ ] Node.js 18.0.0以上がインストールされている
  ```bash
  node --version
  ```
- [ ] npm 9.0.0以上がインストールされている
  ```bash
  npm --version
  ```
- [ ] Gitがインストールされている
  ```bash
  git --version
  ```

## セットアップ手順

### Step 1: 依存関係のインストール

- [ ] `code`ディレクトリに移動
  ```bash
  cd code
  ```
- [ ] 依存関係をインストール
  ```bash
  npm install
  ```
- [ ] インストールが正常に完了したことを確認

### Step 2: 環境変数の設定

- [ ] 環境変数テンプレートをコピー
  ```bash
  cp env.template .env.local
  ```
- [ ] `.env.local`ファイルを開いて編集
- [ ] `DATABASE_URL`をXREAサーバーの接続情報に設定
- [ ] `NEXTAUTH_SECRET`を生成して設定
  ```bash
  openssl rand -base64 32
  ```
- [ ] 必要に応じてAI API Keysを設定（オプション）

### Step 3: 開発サーバーの起動確認

- [ ] 開発サーバーを起動
  ```bash
  npm run dev
  ```
- [ ] ブラウザで `http://localhost:3000` にアクセス
- [ ] ページが正常に表示されることを確認

### Step 4: Prismaクライアントの生成

- [ ] Prismaクライアントを生成
  ```bash
  npm run db:generate
  ```
- [ ] エラーが発生しないことを確認

### Step 5: データベースマイグレーション（データベース接続可能な場合）

- [ ] データベース接続情報が正しく設定されていることを確認
- [ ] 初期マイグレーションを実行
  ```bash
  npm run db:migrate
  ```
- [ ] マイグレーションが正常に完了したことを確認
- [ ] Prisma Studioでデータベースを確認（オプション）
  ```bash
  npm run db:studio
  ```

### Step 6: 初期データの投入（オプション）

- [ ] 初期データを投入
  ```bash
  npm run db:seed
  ```
- [ ] データが正常に投入されたことを確認

## トラブルシューティング

### 問題: `npm install`でエラーが発生

**解決方法**:
1. Node.jsのバージョンを確認（v18以上であること）
2. `package-lock.json`を削除して再実行
3. `node_modules`を削除して再実行

### 問題: 開発サーバーが起動しない

**解決方法**:
1. ポート3000が使用されていないか確認
2. エラーメッセージを確認
3. `.env.local`ファイルが正しく設定されているか確認

### 問題: データベース接続エラー

**解決方法**:
1. `.env.local`の`DATABASE_URL`を再確認
2. XREAサーバーの接続情報を確認
3. 外部接続が許可されているか確認

## 完了確認

すべてのステップが完了したら、以下を確認:

- [ ] 開発サーバーが正常に起動している
- [ ] `http://localhost:3000` でページが表示される
- [ ] エラーが発生していない
- [ ] Prismaクライアントが生成されている（データベース接続可能な場合）
