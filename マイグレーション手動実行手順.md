# マイグレーション手動実行手順

## 📋 現在の状況

Prismaの`db push`コマンドで「Invalid default value for 'createdAt'」エラーが発生しています。

## 🔍 原因

MySQL 8.0では、`DATETIME`型にデフォルト値として`CURRENT_TIMESTAMP`を設定する際に、括弧を使用する必要がある場合があります。

## ✅ 解決方法

### オプション1: マイグレーションファイルを手動で作成・実行

Prismaが生成するSQLを手動で修正して実行する方法：

1. マイグレーションファイルを作成
2. SQLを修正（`DEFAULT CURRENT_TIMESTAMP`を`DEFAULT (CURRENT_TIMESTAMP)`に変更）
3. 手動で実行

### オプション2: 一時的に`@default(now())`を削除

Prismaスキーマから一時的に`@default(now())`を削除して、マイグレーション後にデフォルト値を設定する方法。

### オプション3: 別のアプローチ

開発環境では、テーブルを手動で作成して、Prismaスキーマと同期させる方法もあります。
